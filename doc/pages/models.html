Models

Pages may contain information that are changeable and as such, must be updated on the user's device. To this end, data are being received from and sent to the server. These data are referred to as models.

Every model must have a unique id which differentiates it from others e.g. users. This is required to keep updating working fluently without interfering with other models that are not changed.

Model Attributes

Model attributes can be rendered by putting the key in double curly brackets, that is, {{KEY}}.

For the object below:

var user = { firstName: "John", lastName: "Doe", gender: "Male", hobbies: [ "eating", "sleeping" ], socials: { facebook: "http://facebook.com/john.doe", twitter: "http://twitter.com/john.doe" } }

to show the user's first name, we would do {{firstName}}.

To target hobbies, we would do {{hobbies.0}} and {{hobbies.1}} which would render 'eating' and 'sleeping' respectively.

To target socials, we would do {{socials.facebook}} and {{socials.twitter}}.

Alternatively, we could loop over both hobbies and socials with tag attribute this-each

Filtering

Model attributes can be filtered by appending the desired filters the the attributes name, all separated by pipes (|). e.g. {{firstName|trim|capitalize}}. The value of `firstName` is passed to filters left-to-right, that is to trim first, then to capitalize.
