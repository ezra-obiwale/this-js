Pages

Every application is split into pages. And each application can have as many pages as required.
Every page must have a unique id.

Default Page

To specify a page as the default page, add attribute `(!this-default-page)`

Server-side HTML

Page contents may be loaded dynamically from the server. To do this, add attribute `this-url` with the value as the URL.

Optionally, you could go structural.

With a Layout

If a page should use a layout, add attribute `this-layout` with the name of the layout as the value: 

<page this-id="home" this-layout="main">...</page>

Events

The following custom events are available on pages:

Event Type Description
page.leave Fires when the current page is about to be left and another page about to be loaded
page.load.canceled Fires when an attempted page load was canceled
page.load.failed This is triggered when a page was found but could not be loaded.
page.loaded Fires when every component, model, form, collection, etc on the page has been loaded
